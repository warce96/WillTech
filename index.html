Resumen Ejecutivo

Willtech propone el desarrollo de un sitio web para MotoYa que permita a los clientes crear pedidos de delivery de forma rápida desde el celular, con opción de cobro online (Dinelco / uPay / Bancard) o pedido sin cobro. Del lado de MotoYa, se incluye un panel de administración para visualizar y gestionar todos los pedidos con sus datos completos.

Objetivo del Proyecto

  • Facilitar que el cliente cree un pedido en menos de 2 minutos.
  • Reducir errores de dirección y datos del pedido.
  • Centralizar pedidos en un panel para el equipo de MotoYa.
  • Habilitar pago online opcional cuando el pedido sea “cobrado”.

Alcance Funcional

1) Portal del Cliente (Web)

  • Crear pedido de delivery (formulario).
  • Ingresar dirección de retiro (origen).
  • Ingresar dirección de entrega (destino).
  • Seleccionar si el pedido se cobra o no.
  • Si “se cobra”, habilitar pasarela/túnel de pago (Dinelco / uPay / Bancard).
  • Describir el paquete / detalles del envío.
  • Ingresar nombre y WhatsApp del cliente.
  • Confirmación del pedido + mensaje por WhatsApp (opcional).

Nota: El “túnel de pago” se integra según disponibilidad de API/credenciales del proveedor de pago.

2) Panel Administrativo (MotoYa)

  • Login de administrador.
  • Listado de pedidos con filtros (fecha, estado, cobrado/no cobrado).
  • Detalle completo del pedido (origen, destino, paquete, cliente, WhatsApp).
  • Gestión de estados: Nuevo → En proceso → Entregado / Cancelado.
  • Registro de pago: estado del pago (pendiente / aprobado / rechazado).
  • Exportación básica (CSV/Excel) opcional.

Flujo de Pedido (Cliente)

  1. Cliente ingresa a la web de MotoYa.
  2. Completa retiro, entrega, descripción del paquete, nombre y WhatsApp.
  3. Selecciona “¿Se cobra?”:
    • Si No → confirma pedido y se registra en el panel.
    • Si Sí → se muestra el pago (Dinelco/uPay/Bancard) y, al aprobarse, se confirma.
  4. MotoYa visualiza el pedido en el panel y gestiona el estado.

Integración de Pagos

La integración del pago se realizará con el proveedor que el cliente defina (Dinelco, uPay o Bancard), sujeto a la disponibilidad de su documentación técnica y credenciales.

  • Opción “cobrado”: el cliente paga antes de confirmar el pedido (flujo recomendado).
  • Registro del estado del pago en el panel.
  • Webhooks/confirmación automática si el proveedor lo permite.

Si el proveedor requiere alta comercial previa o contratos, ese proceso queda a cargo de MotoYa.

Requisitos / Insumos del Cliente

  • Logo e identidad visual de MotoYa (o lineamientos de colores/tipografía).
  • Dominio (ej. motoya.com.py) y acceso al hosting/VPS (si aplica).
  • Credenciales/documentación de la pasarela de pago elegida (Dinelco/uPay/Bancard).
  • Textos básicos: contacto, horarios, cobertura, políticas (opcional).
